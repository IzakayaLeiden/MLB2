# model-v3 회고 평가

## 결론

`model-v3`는 선발투수 준비 상태, 타선, 핵심 불펜 가용성, 구장·일정 상태를 추가해 `model-v1`과 `model-v2`보다 2025 전체 경기 적중률과 확률 지표를 높였다. 그러나 60%에는 도달하지 못했고 Elo보다 적중률도 낮으므로 배포 모델로 승격하지 않는다.

- 선택 후보: `starter_readiness_lineup_reliever_schedule_interactions`, L2 `0.1`, 전체 과거 시즌 학습
- 2025 적중률: **55.73%** (`1,352 / 2,426`)
- 60% 도달에 필요한 추가 정답: **104경기**
- Elo: **56.14%** (`1,362 / 2,426`)
- `model-v1`: **55.28%** (`1,341 / 2,426`)
- `model-v2`: **55.40%** (`1,344 / 2,426`)
- 승격 여부: **불가**

이 결과는 `2025`를 이미 여러 차례 확인한 개발 홀드아웃 결과다. 새로운 봉인 홀드아웃이나 미래 검증 결과로 표현하면 안 된다.

## 추가한 정보

모든 수치형 상태는 대상 경기의 공식 날짜보다 이전 경기만 사용한다.

- 선발투수 휴식일
- 이전 시즌과 최근 5번 선발 등판을 혼합한 예상 이닝
- 경기 타선의 과거 시점 평활 OPS 및 상위 4명 OPS
- 과거 역할과 성적을 기준으로 고른 핵심 구원투수 4명의 최근 1~3일 투구량
- 핵심 불펜의 품질, 피로, 사용 불가 추정 인원
- 구장별 과거 홈 승리 편향과 득점 환경
- 팀별 연속 경기일, 무휴 구장 이동, 야간 후 주간 경기, 원정 연속 경기 수
- Elo, 선발 예상 이닝, 선발 성적, 타선, 불펜 사이의 사전 정의된 상호작용

같은 공식 날짜의 기록은 사용하지 않으므로 더블헤더 1차전 결과가 2차전 입력에 들어가지 않는다.

## 모델 선택 절차

후보는 2022, 2023, 2024 워크포워드 예측만으로 선택했다. 후보에는 피처 블록, L2 값, 최근 3·4·5시즌 학습창과 전체 학습창이 포함된다. 순수 NumPy LDA 분류기와 최근 공격·수비를 분리한 후보도 같은 규칙으로 비교했지만 선택되지 않았다.

추가로 최근 3회 선발 흐름, 타자·투수 손잡이 구성, 이전 시즌 좌우 상대 OPS, 완료 경기 득실점 EWMA, 구장·일정 상태를 각각 비교했다. MLB `statSplits` API는 과거 날짜 범위를 적용하지 않으므로 좌우 상대 성적은 대상 시즌의 이전 시즌 값만 사용했다. Elo와 회귀 확률을 25%·50%·75% 비율로 섞은 후보도 비교했지만 선택되지 않았다. 구장·일정 결합 후보는 기존 후보와 최저 시즌 정확도가 같고 합산 정확도가 높아 선택됐다.

| 개발 후보 | 2022~2024 합산 | 최저 시즌 | 선택 |
|---|---:|---:|---:|
| 기존 선발·타선·불펜 상호작용 | 58.04% | 57.21% | 이전 선택 |
| 구장·일정 결합 | 58.20% | 57.21% | 선택 |
| 최근 선발 흐름 | 57.50% | 56.62% | 탈락 |
| 타자·투수 손잡이 구성 | 57.93% | 57.13% | 탈락 |
| 이전 시즌 좌우 상대 OPS | 57.86% | 56.95% | 탈락 |
| 동적 공격·수비 득실점 강도 | 58.05% | 56.97% | 탈락 |

한 시즌에서만 크게 좋아지는 후보를 피하기 위해 다음 순서로 정렬한다.

1. 2022~2024 중 최저 시즌 적중률이 높은 후보
2. 세 시즌 합산 적중률이 높은 후보
3. Log Loss가 낮은 후보
4. Brier Score가 낮은 후보

최근 5시즌 학습 후보는 개발 구간 합산 적중률이 가장 높았지만 2025 성능이 크게 떨어졌다. 이는 시간창 추가가 안정적인 개선이 아니라 후보 선택 과적합일 수 있음을 보여준다.

## 시즌별 결과

| 시즌 | 경기 | Elo | model-v1 | model-v2 | model-v3 |
|---|---:|---:|---:|---:|---:|
| 2022 | 2,429 | 57.60% | 57.84% | 60.15% | 59.90% |
| 2023 | 2,423 | 55.51% | 55.55% | 56.29% | 57.49% |
| 2024 | 2,426 | 55.52% | 54.99% | 56.51% | 57.21% |
| 2025 개발 홀드아웃 | 2,426 | 56.14% | 55.28% | 55.40% | 55.73% |

2025의 확률 지표는 다음과 같다.

| 모델 | Log Loss | Brier Score | Accuracy |
|---|---:|---:|---:|
| Elo | 0.687604 | 0.247117 | 56.14% |
| model-v1 | 0.681911 | 0.244495 | 55.28% |
| model-v2 | 0.681538 | 0.244363 | 55.40% |
| model-v3 | 0.679517 | 0.243368 | 55.73% |

## 통계적 해석

2025 날짜 블록 10,000회 부트스트랩에서 model-v3의 적중률 개선은 통계적으로 확정되지 않았다.

- model-v1 대비 오류율 차이: `-0.45%p`, 95% CI `[-2.32%p, +1.37%p]`, p=`0.6346`
- model-v2 대비 오류율 차이: `-0.33%p`, 95% CI `[-1.82%p, +1.16%p]`, p=`0.6702`
- Elo 대비 오류율 차이: `+0.41%p`, 95% CI `[-1.45%p, +2.19%p]`, p=`0.6546`

model-v3의 Elo 대비 Log Loss와 Brier Score는 유의하게 낮았다(`p=0.0106`, `p=0.0130`). 적중률은 그렇지 않았다. 따라서 “확률 배분은 더 적절하지만 승자를 더 잘 맞힌다고 입증되지는 않았다”가 정확한 설명이다.

## 왜 전체 경기 60%가 아닌가

2025에서 model-v3와 Elo가 같은 팀을 선택한 1,886경기의 적중률은 57.64%였다. 서로 다른 팀을 선택한 540경기에서는 model-v3가 49.07%, Elo가 50.93%였다. 새 피처가 Elo의 어려운 결정을 안정적으로 뒤집지 못했다.

확신도에 따라서도 차이가 크다.

- 전체 2,426경기: 55.73%
- 확률이 50%에서 10%p 이상 떨어진 568경기: **63.38%**, 커버리지 23.4%

따라서 현재 모델로 60%를 표시하려면 약 23%의 고확신 경기만 선택해야 한다. 이는 전체 경기 적중률 60%가 아니며 사이트에서 반드시 커버리지를 함께 밝혀야 한다.

## 남은 핵심 병목

1. 과거 선발과 타선 ID에는 실제 경기 전 관측 시각이 없다. 회고 API 응답을 사용했으므로 prospective snapshot이 필요하다.
2. 선택 모델의 타선은 단순 평활 OPS다. 손잡이 구성과 이전 시즌 좌우 상대 OPS 후보를 시험했지만 개발 구간에서 개선되지 않았고, 예상 타석·수비·결장 영향은 아직 충분히 반영하지 않는다.
3. 선발은 K-BB, 실점, 홈런과 예상 이닝만 사용한다. 구종, 구속, Stuff, 부상 복귀와 투구 수 제한이 없다.
4. 불펜은 핵심 4명의 휴식 규칙 기반 추정이다. 당일 로스터 이동과 실제 가용 여부를 알지 못한다.
5. 구장별 과거 환경과 일정 피로는 추가했지만, 실제 이동 거리·시간대, 날씨, 돔 상태와 경기 종료 시각은 없다.
6. 2025는 더 이상 봉인되지 않았다. 새로운 설계의 일반화 여부는 동결 이후 경기로 확인해야 한다.

## 다음 버전의 최소 요구사항

- 경기 시작 60분 전 타임스탬프가 있는 선발·라인업·로스터 snapshot
- 좌우 매치업 기반 타선 지표와 투수 예상 이닝 모델
- 구원투수별 실제 가용 확률
- 외부 파크 팩터, 날씨, 실제 이동 거리·시간대
- 현 선형 모델과 비선형 모델의 동일 워크포워드 비교
- 전체 경기 Accuracy와 `accuracy at coverage`를 분리한 사전 승격 기준
- 새 동결 시점 이후의 독립적인 미래 평가

## 경기 전 섀도 스냅샷

`snapshot-model-v3`의 `pregame-model-v3-snapshot-v6`는 다음 항목을 경기 시작 60분 전 기준으로 봉인한다.

- 당일 probable starter ID
- 확정 타선 또는 전날까지의 가장 최근 완료 경기 기반 예상 타선
- 당일 active roster의 투수 ID
- 대상 공식 날짜 이전의 선발, 타자, 구원투수 기록
- 선발 휴식·예상 이닝·최근 3회 흐름, 타선 OPS·손잡이·이전 시즌 좌우 상대 OPS, 핵심 불펜 품질·피로·가용성 피처
- 대상 날짜 이전 완료 경기만 누적한 동적 공격·수비 득실점 강도
- 대상 날짜 이전 완료 경기만 누적한 구장 환경과 팀 일정 피로
- 각 API 응답의 수집 시각, URL, SHA-256

라인업이 없고 과거 완료 경기에서도 예상 타선을 만들 수 없거나, active roster 및 핵심 구원투수 4명을 확인할 수 없거나, 어떤 소스든 시작 60분 전 컷오프보다 늦으면 스냅샷 품질은 실패한다. 같은 경기 ID의 기존 파일은 덮어쓰지 않는다.

2026-07-21 owner-only 실제 스모크 결과는 다음과 같다.

- 예정 경기 14개 모두 품질 통과
- 확정 타선 14팀, 최근 완료 경기 기반 예상 타선 14팀
- 14경기 모두 양 팀 핵심 구원투수 4명 확인
- 14경기 모두 0이 아닌 과거 누적 득실점 강도 확인
- 14경기 모두 구장 이력·원정 연속 경기 상태 확인
- 최신 소스 수집 시각 `2026-07-21T20:32:54Z`

이 스모크는 데이터 시점과 운영 무결성만 확인한다. 예측 성능을 검증하거나 model-v3 승격을 허용하지 않는다. daily shadow workflow는 날짜별 `model-v3-YYYY-MM-DD.zip`을 비공개 draft release에 저장한다.

## 재현

```powershell
$env:PYTHONPATH = "src"
python -m mlb_predictor backtest-model-v3 `
  --features data\multi-year\features\pregame_features.parquet `
  --games data\multi-year\processed\games.parquet `
  --output-dir data\model-v3-retrospective `
  --cache-dir data\pitching-backtest-cache `
  --bootstrap-iterations 10000

python -m mlb_predictor snapshot-model-v3 `
  --history-games data\multi-year\processed\history_games.parquet `
  --target-date 2026-07-21 `
  --output-dir data\model-v3-snapshots `
  --cache-dir data\model-v3-snapshot-cache
```

상세 후보, 시즌별 지표, 부트스트랩 결과는 생성 파일 `data/model-v3-retrospective/evaluation.json`에 기록된다. `data/`는 생성 데이터이므로 Git에 추적하지 않는다.
