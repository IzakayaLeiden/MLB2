# MLB2 model-v1 기술 설명 및 외부 평가 가이드

문서 기준일: 2026-07-21  
대상 모델: `model-v1`  
모델 정규 payload SHA-256: `905a6f001a2f03649cba49027e00ada38c5b3c9bd29e008c953a7c182494bf21`  
선택 fingerprint: `2a09b40640b1dc5903fa2c41b8dc45ad90eb524d9788c3882c602ebc22a9852e`

이 문서는 MLB2가 경기 시작 전에 홈팀 승리확률을 계산하는 방식을 제3자가 독립적으로 검토할 수 있도록 작성한 모델 카드다. 구현을 정당화하기 위한 홍보 문서가 아니며, 확인된 성능과 한계를 함께 기록한다.

## 1. 요약 판단

`model-v1`은 Elo와 최근 팀 상태를 입력으로 사용하는 **Platt 보정 L2 로지스틱 회귀**다. 2022~2024 워크포워드에서 후보와 L2 값을 선택한 뒤, 선택 과정에서 사용하지 않은 2025년 정규시즌 2,426경기를 한 번만 평가했다.

2025 봉인 홀드아웃에서 `model-v1`은 상수 기준선과 Elo보다 Log Loss와 Brier Score가 모두 낮았다. 그러나 0.5 임계값 적중률은 55.28%로 Elo의 56.14%보다 낮았다. 따라서 이 모델을 “승자를 가장 많이 맞히는 모델”로 해석하면 안 된다. 현재 선택 근거는 **확률 예측의 평균 품질과 보정**이다.

이 모델은 누수 방지와 재현성 측면에서는 검토 가능한 v1이지만, 선발투수 성적·라인업·부상·불펜 소모·날씨를 사용하지 않으며 성능 개선 폭도 작다. 독립 평가자는 실제 활용 가능성을 판단하기 전에 아래의 한계와 감사 체크리스트를 확인해야 한다.

## 2. 예측 대상과 범위

- 대상: MLB 정규시즌 경기의 홈팀 승리확률
- 타깃: `home_win` (`1`이면 홈팀 승리, `0`이면 원정팀 승리)
- 제외: 포스트시즌, 무승부, 취소 경기, 완료되지 않은 경기, 점수 누락 경기
- 시간대: 일정 및 당일 구분은 `America/New_York`
- 사용 목적: 정보 제공 및 모델 검증
- 비목표: 배당 비교, 베팅 추천, ROI 또는 수익성 추정

선발투수 이름은 화면 표시용으로만 보존된다. 선발투수의 이름이나 성적은 모델 입력에 포함되지 않는다.

|---|---:|

| 완료 정규시즌 경기 | 19,379 |
| 팀 | 30 |
| 원시 로지스틱 학습 행 | 17,875 |
| Platt 보정 행 | 1,504 |
| 데이터 품질 Critical/High 오류 | 0 |

최종 동결 모델은 2018~2025 데이터를 원시 로지스틱 회귀 학습에 사용하고, 2026년 7월 20일까지 완료된 1,504경기를 Platt 보정에 사용한다. 전체 입력 행은 19,379개다.

데이터 원천은 MLB Stats API 일정 응답이다. 이 API 응답 구조는 이 저장소가 통제하는 안정적 계약이 아니므로 원본 캐시, 정규화 및 품질 보고서를 함께 검토해야 한다.

### 3.2 시점 정책과 누수 차단

피처는 `prior_official_date_only` 정책으로 생성한다.

1. 한 공식 날짜에 열리는 모든 경기의 피처를 그 날짜가 시작되기 전 상태로 계산한다.
2. 그 날짜의 모든 피처를 만든 뒤에만 경기 결과를 팀 상태와 Elo에 반영한다.
3. 따라서 더블헤더 1차전 결과도 같은 공식 날짜의 2차전 피처에 들어가지 않는다.
4. 미래 경기에는 결과 타깃이나 점수 필드를 만들지 않는다.

이 정책은 실제 시작 시각보다 보수적이다. 같은 날 먼저 끝난 경기의 정보를 나중 경기에도 사용하지 않는다.

## 4. 팀 상태 계산

### 4.1 Elo

팀은 최초 `1500`점에서 시작한다. 홈팀의 Elo 기대승률은 다음과 같다.

```text
p_elo = 1 / (1 + 10 ^ (-((Elo_home + 35) - Elo_away) / 400))
```

설정값:

| 설정 | 값 |
|---|---:|
| 초기 Elo | 1500 |
| K-factor | 20 |
| 홈 어드밴티지 | 35 Elo 점 |
| 오프시즌 전년도 Elo 유지율 | 0.75 |

경기 후 변화량은 홈팀 기준 `20 × (실제 결과 - 기대승률)`이다. 오프시즌에는 1500을 중심으로 전년도 편차의 75%만 다음 시즌에 유지한다.

### 4.2 시즌 및 최근 상태

- 시즌 승률과 경기 수는 시즌이 바뀌면 초기화한다.
- 최근 성적과 득실점은 최근 10경기 이동 창을 사용한다.
- 이전 경기가 없으면 승률 0.5, 경기당 득점·실점 4.5, 휴식일 5일의 중립값을 사용한다.
- 휴식일은 현재 공식 날짜와 마지막 경기 날짜 사이의 비경기 일수다.

## 5. 모델 입력 피처

모델은 다음 12개 입력을 순서대로 사용한다.

| 피처 | 정의 |
|---|---|
| `home_elo_minus_away` | 홈 Elo - 원정 Elo |
| `season_win_pct_difference` | 홈 시즌 승률 - 원정 시즌 승률 |
| `recent_win_pct_difference` | 홈 최근 10경기 승률 - 원정 최근 10경기 승률 |
| `recent_run_margin_difference` | 홈 최근 경기당 득실차 - 원정 최근 경기당 득실차 |
| `rest_days_difference` | 홈 휴식일 - 원정 휴식일 |
| `home_games_before` | 홈팀의 해당 시즌 이전 경기 수 |
| `away_games_before` | 원정팀의 해당 시즌 이전 경기 수 |
| `home_recent_games_count` | 홈 최근 창에 실제로 존재하는 경기 수 |
| `away_recent_games_count` | 원정 최근 창에 실제로 존재하는 경기 수 |
| `home_has_prior_history` | 홈팀의 해당 시즌 이전 경기 존재 여부 |
| `away_has_prior_history` | 원정팀의 해당 시즌 이전 경기 존재 여부 |
| `is_night_game` | 일정의 `day_night` 값이 `night`이면 1 |

포함되지 않는 주요 정보:

- 선발투수 및 불펜 성적
- 확정 라인업과 선수 부상
- 구장별 파크 팩터
- 날씨와 이동 거리
- 시장 배당

## 6. 확률 계산

### 6.1 표준화 로지스틱 회귀

각 피처 `x_j`는 원시 학습 집합의 평균 `μ_j`와 표준편차 `σ_j`로 표준화한다.

```text
z_j = (x_j - μ_j) / σ_j
s_raw = intercept + Σ(β_j × z_j)
p_raw = sigmoid(s_raw)
```

절편에는 벌점을 적용하지 않고, 나머지 계수에는 L2 벌점 `0.01`을 적용한다. 최적화는 Newton 방식과 backtracking line search를 사용한다.

최종 동결 계수는 다음과 같다. 계수는 표준화된 피처에 대한 값이며, 상관된 피처가 함께 들어가므로 인과효과로 해석하면 안 된다.

| 피처 | 평균 `μ` | 표준편차 `σ` | 계수 `β` |
|---|---:|---:|---:|
| `home_elo_minus_away` | 0.262241 | 91.659237 | 0.320975 |
| `season_win_pct_difference` | 0.001422 | 0.189848 | 0.039105 |
| `recent_win_pct_difference` | -0.006466 | 0.287975 | -0.144126 |
| `recent_run_margin_difference` | -0.006191 | 2.639987 | 0.090500 |
| `rest_days_difference` | 0.003804 | 0.362412 | 0.001198 |
| `home_games_before` | 77.775608 | 47.003143 | -0.001858 |
| `away_games_before` | 77.782154 | 46.993012 | -0.007741 |
| `home_recent_games_count` | 9.629203 | 1.562647 | 0.008135 |
| `away_recent_games_count` | 9.630769 | 1.569661 | 0.040090 |
| `home_has_prior_history` | 0.993399 | 0.080980 | 0.040077 |
| `away_has_prior_history` | 0.993175 | 0.082332 | -0.067222 |
| `is_night_game` | 0.635860 | 0.481188 | -0.024880 |

동결 모델 절편은 `0.1298289108`이다.

`recent_win_pct_difference`의 음수 계수처럼 직관과 다른 부호가 존재한다. Elo, 시즌 승률, 최근 승률, 득실차가 서로 상관되어 있기 때문에 단일 계수 부호만으로 피처가 잘못됐거나 역효과라고 단정할 수는 없다. 다만 독립 평가 시 안정성 검토가 필요한 신호다.

### 6.2 Platt 보정

원시 확률에 다음 보정을 적용해 최종 홈 승리확률을 만든다.

```text
p_final = sigmoid(0.03174830196 + 0.5004622693 × logit(p_raw))
```

보정 계수가 1보다 작기 때문에 극단적인 원시 확률을 0.5 방향으로 압축하는 경향이 있다. 보정기의 L2 값은 `0.000001`이다.

## 7. 후보 선택 절차

후보는 다음으로 제한했다.

- 시즌 이전 학습 집합의 홈 승률을 내는 상수 모델
- Elo 단독 모델
- L2 로지스틱 회귀
- Platt 보정 L2 로지스틱 회귀

로지스틱 후보의 L2 값은 `0.01`, `0.1`, `1.0`, `10.0`이다. 2022, 2023, 2024년을 각각 시간 순서대로 검증했고 총 7,278경기를 합산했다. Platt 후보는 각 검증 시즌 직전 시즌을 보정에 사용하고, 그보다 앞선 시즌만 원시 로지스틱 학습에 사용했다.

선택 순서는 다음과 같다.

1. Log Loss가 가장 낮은 후보
2. 동률이면 Brier Score가 더 낮은 후보
3. 그래도 같으면 후보 키의 사전순

선택된 후보는 `logistic_platt:l2=0.01`이다.

### 7.1 2022~2024 워크포워드 결과

| 모델 | 경기 | Accuracy | Log Loss | Brier Score |
|---|---:|---:|---:|---:|
| 상수 | 7,278 | 52.54% | 0.691961 | 0.249407 |
| Elo | 7,278 | 56.21% | 0.687719 | 0.246886 |
| 선택 모델 | 7,278 | 56.13% | **0.679851** | **0.243413** |

Accuracy는 선택 기준이 아니었다.

## 8. 2025 봉인 홀드아웃

2025년 정규시즌 2,426경기는 후보와 L2 선택이 끝날 때까지 평가에 사용하지 않았다. 홀드아웃 평가 코드는 상태 파일의 존재 여부로 재사용을 차단하며, 기록된 사용 횟수는 1회다.

2025 평가용 선택 모델은 2018~2023 데이터를 원시 로지스틱 학습에 사용하고 2024년을 Platt 보정에 사용했다. 따라서 2025 결과는 해당 모델 구조와 선택 절차의 외부 시간 검증이다.

### 8.1 결과

| 모델 | 적중/전체 | Accuracy | Log Loss | Brier Score |
|---|---:|---:|---:|---:|
| 상수 | 1,316 / 2,426 | 54.25% | 0.689853 | 0.248354 |
| Elo | 1,362 / 2,426 | **56.14%** | 0.687604 | 0.247117 |
| 선택 모델 | 1,341 / 2,426 | 55.28% | **0.681911** | **0.244495** |

통과 조건:

- 선택 모델의 Log Loss와 Brier가 상수보다 모두 낮아야 한다.
- 선택 모델이 LR 계열이면 두 지표가 Elo보다도 모두 낮아야 한다.

선택 모델은 이 조건을 통과했다. 반면 0.5 임계값 Accuracy는 Elo보다 0.87%p 낮다.

현재 보고서에는 부트스트랩 신뢰구간, Diebold-Mariano 유형 비교, 시즌·팀별 오차 분석 또는 다중 후보 선택에 대한 보정이 없다. 따라서 작은 지표 차이가 통계적으로 안정적인지 별도 검증이 필요하다.

## 9. 최종 동결 모델과 홀드아웃 모델의 관계

2025 홀드아웃 통과 후 선택된 **구조와 L2 값**을 유지한 채 2026-07-20까지 데이터로 다시 학습해 `model-v1`을 동결했다.

따라서 다음 두 모델은 동일한 설정을 사용하지만 계수는 동일하지 않다.

- 2025 홀드아웃 평가 모델: 2018~2023 원시 학습 + 2024 보정
- 현재 동결 모델: 2018~2025 원시 학습 + 2026-07-20까지 보정

2025 홀드아웃 지표는 모델 선택 절차와 구조의 검증 결과이지, 2025 데이터를 포함해 재학습된 최종 계수 자체를 동일한 2025 데이터에서 다시 검증한 결과가 아니다. 최종 계수의 실제 미래 성능은 2026년 봉인 예측을 누적해 별도로 평가한다.

## 10. 당일 예측 흐름

```text
전날까지 완료 경기
  -> 날짜 단위 팀 상태와 Elo 갱신
  -> 당일 예정 경기 피처 생성
  -> 동결 모델 SHA-256 확인
  -> 표준화 로지스틱 확률 계산
  -> Platt 보정
  -> 경기 시작 60분 전까지 prediction-feed-v1 봉인
```

운영 규칙:

- 생성 시각의 ET 날짜와 예측 대상 날짜가 같아야 한다.
- 경기 시작 60분 전을 지난 예측은 제외하며 품질 상태를 실패로 만든다.
- 같은 경기의 기존 봉인 예측은 덮어쓰지 않는다.
- 예측 파일은 이전 파일 SHA-256을 포함하는 해시 체인으로 연결한다.
- 결과는 예측 파일에 넣지 않고 별도 `prediction-grade-v1` 파일로 연결한다.
- 피드가 오래됐거나 날짜·스키마·품질 검사가 실패하면 사이트는 확률을 숨긴다.

## 11. 알려진 한계와 위험

1. **투수 정보 부재**: MLB 경기에서 영향이 큰 선발투수와 불펜 상태를 사용하지 않는다.
2. **낮은 절대 적중률**: 2025 Accuracy는 55.28%이며 Elo보다 낮다.
3. **작은 확률 점수 개선**: Elo 대비 개선 폭은 Log Loss 약 `0.00569`, Brier 약 `0.00262`다.
4. **통계적 불확실성 미보고**: 지표 차이의 신뢰구간과 유의성 검정이 없다.
5. **단일 봉인 시즌**: 완전히 봉인된 최종 평가는 2025년 한 시즌뿐이다.
6. **API 계약 위험**: 데이터 원천의 응답 구조가 변경될 수 있다.
7. **피처 상관성**: 계수의 크기와 부호를 독립적 영향 또는 인과관계로 해석할 수 없다.
8. **부분 시즌 보정**: 최종 모델의 Platt 보정은 2026년 7월 20일까지의 부분 시즌을 사용한다.
9. **팀·구간별 성능 미보고**: 팀, 월, 확률 구간, 더블헤더, 경기 시간대별 상세 오차가 아직 없다.
10. **경제적 유용성 미검증**: 시장 배당을 사용하지 않으므로 수익성이나 시장 대비 우위를 주장할 수 없다.

## 12. 외부 평가자 체크리스트

### 데이터와 누수

- [ ] 원본 일정 캐시에서 완료 경기와 제외 사유를 표본 감사한다.
- [ ] `official_date`보다 같은 날 또는 미래인 결과가 피처에 들어가지 않는지 확인한다.
- [ ] 더블헤더가 동일한 사전 스냅샷을 사용하는지 재현한다.
- [ ] 2018~2026 팀 ID 변경 및 팀 커버리지를 확인한다.

### 모델과 재현성

- [ ] `selection.json`에서 모든 후보의 순위가 Log Loss/Brier 정책과 일치하는지 확인한다.
- [ ] 2025 홀드아웃 상태 파일의 단회 사용 제약을 확인한다.
- [ ] `model-v1.json`의 정규화 통계, 계수, 보정 계수 및 해시를 독립 구현과 비교한다.
- [ ] 같은 입력 행에서 Python 구현과 독립 구현의 확률이 일치하는지 확인한다.

### 성능

- [ ] 시즌 또는 날짜 블록 부트스트랩으로 지표 차이의 신뢰구간을 계산한다.
- [ ] 팀·월·확률 구간별 calibration과 오차를 확인한다.
- [ ] Accuracy뿐 아니라 Log Loss, Brier 및 calibration curve를 함께 평가한다.
- [ ] Elo보다 복잡한 모델이 실제로 안정적인 개선을 제공하는지 검정한다.
- [ ] 2026 미래 봉인 예측만으로 별도 성능을 평가한다.

### 운영

- [ ] 경기 시작 후 생성된 예측이 없는지 생성 시각과 경기 시각을 대조한다.
- [ ] 동일 경기 예측 덮어쓰기 방지와 해시 체인을 확인한다.
- [ ] 데이터 노후화·부분 피드·품질 실패 시 화면이 fail-closed인지 확인한다.

## 15. 결론

`model-v1`은 시점 누수를 차단한 과거 데이터로 구축되고 2025 봉인 홀드아웃에서 확률 점수 기준을 통과한 보수적 기준 모델이다. 현재 근거는 “확률 품질이 상수와 Elo보다 소폭 개선됐다”는 주장까지 지지한다. 높은 적중률, 시장 대비 우위, 수익성 또는 투수·라인업 상황까지 반영한 종합 예측력은 지지하지 않는다.

외부 평가의 우선순위는 지표 차이의 통계적 안정성, 2026 미래 봉인 성능, 부분 시즌 보정의 안정성, 그리고 투수·불펜 피처 추가가 시간 외 검증에서 실제 개선을 만드는지 확인하는 것이다.
